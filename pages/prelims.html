Hypertext Transfer Protocol (HTTP)
  • Application Layer Communication Protocol used to access resources on the world wide web. Jointly developed by the W3C and IETF (INTERNET ENGINEERING TASK FORCE).

Protocol
  • is a Set of rules that end points in a telecommunication connection use when they communicate.

History of HTTP
  • HTTP 0.9 (1991)
  • HTTP 1.0 (RFC 1945 - MAY 1996) – single line, resource, request, and response, after that, it terminates the line.
  • HTTP 1.1 (RFC 2068 - JAN 1997, RFC 2616 – JUNE 1999, RFC 7230-7235 – JUNE 2014).
  • HTTP 2 (RFC 7540 – MAY 2015) - speeding the communication client to server.

HTTP Fundamentals
  • HTTP typically runs on top of TCP/IP using TCP port 80 by default, or TCP port 443 for HTTPS.
  • HTTP is based on client-server architecture.
  • Clients, aka user agents (UA):
    • Web browsers, web crawlers, email clients, other end user tools and applications
  • Servers:
    • Origin servers, proxy servers, gateways, tunnels
  • HTTP uses a request-response standard protocol
    • The client sends an HTTP request message to the server.
    • The server processes the request and replies with an HTTP response message.
  • HTTP is a stateless communications protocol
    • Servers do not keep information about clients in between requests.
    • Web applications effect session tracking using mechanism such as cookies on URL-encoded session information to keep track of related client requests.
  • HTTP provides support for other functionalities such as cache control, content media type (MIME) specification, language and character set specification, content/transfer coding, client-server protocol negotiations, persistent connections, request pipelining, etc.
  • HTTP resources are identified using URIs (specifically, HTTP URLs)
    • Scheme (http: or https:)
    • (optional)authentication information
    • Host and (optional)port number
    • Path (resolved to the document root on the server) to the resource
    • (optional) scheme-specific parameters
    • (optional) URL-encoded query
    • (optional) bookmark(or fragment identifier)

HTTP Resource Addressing

  • HTTP resources are identified using URIs (RFC 3986), or more specifically, HTTP URLs.
    • Scheme (http: or https:)
    • (optional) authentication information
    • Host and (optional) port number
    • Path (resolved to the document root on the server) to the resource
    • (optional) scheme-specific parameters
    • (optional) URL-encoded query
    • (optional) bookmark (or fragment identifier)

    example for HTTP RESOURCE ADDRESS

    http://usr:pwd@server.org:8080/info/profile.php?id=1234#addr

    • Scheme: http
    • Credentials: usr:pwd
    • Domain Name: @server.org
    • Port Number: 8080
    • Path To Reserouce: /info/profile.php
    • Query: ?id=1234
    • Fragment Identifier: #addr

HTTP Request Methods
  • GET
    • Most commonly used HTTP method.
    • Used to request from the server the retrieval of the source identified by the request URI; the retrieved resource is returned in the message body as an entity.
    • Can be combined with conditional and/or range request headers to effect conditional and/or partial resource retrieval.
    • Must be supported by all general-purpose servers.
  • HEAD
    • Identical to GET, except the message body is not included in the response.
  • POST
    • Request that the origin server accept the entity enclosed in the request as a new subordinate of the resource identified by the Request-URI in the Request-Line.
  • OPTIONS
    • Request for information about the communication options available on the request/response chain identified by the Request-URI. This method allows the client to determine the options and/or requirements associated with a resource, or the capabilities of a server, without implying a resource action or initiating resource retrieval.
  • TRACE
    • Request the server to “echo” back to the client the received request.
    • Typically used for testing/diagnostics of the request chain.
  • PUT
    • Request the server to store the enclosed entity in the message under the specified request URI.
  • DELETE
    • Request the server to delete the resource identified by the request URI.
  • CONNECT
    • Reserved for use of tunneling proxy servers.
  • Idempotent Methods
    • The methods GET, HEAD, PUT, DELETE, TRACE, and OPTION are element of a set that is unchanged in value when multiplied or otherwise operated on by itself.
  • Safe Methods
    • The methods POST, PUT, and DELETE are element of a set that has a change in value when multiplied or otherwise operated on by itself.
  • Unsafe Methods
    • The methods GET, HEAD, TRACE, and OPTION are element of a set that has no change in value when multiplied or otherwise operated on by itself.

HTTP Message Headers
  • GENERAL HEADER FIELDS
    • uses both client and the server.
      • Cache-control: cache response server to client.
      • Connection: client to server/network to network.
      • Date: response created.
      • Pragma: extra communication.
      • Trailer: additional information/message header at the end of the line.
      • Transfer-encoding: chincked encoding. i.e size of the encoded file/data.
      • Upgrade: upgrading applicaiton/information.
      • Via: coming from other information.
      • Warning: protocol variation.
  • REQUEST HEADER FIELDS
    • it comes from the client's request.
      • Accept
      • Accept-charset
      • Accept-encoding
      • Accept-Language
      • Authorization
      • Expect
      • From
      • Host
      • If-Match
      • If-Modified-Since
      • If-None-Match
      • If-Range
      • If-Unmodified-Since
      • Max-forward
      • Proxy-Authorization
      • Range
      • Referer
      • User-Agent
  • RESPONSE HEADER FIELDS
    • it comes from the server's response.
      • Accept-Range: portion of the copy accepted.
      • Age: age/generation of the resource.
      • E-Tag: calculate the size of the resource.
      • Location: redirection of the resource.
      • Proxy-Authenticate
      • Retry-After: service unavailable/under maintenance.
      • Server: information about the server.
      • Vary
      • WWW-Authenticate
  • ENTITY HEADER FIELDS
    • message body.
      • Allow
      • Content-encoding
      • Content-language
      • Content-length
      • Content-location: fetching the location of the resource.
      • Content-MD5: messege digest.
      • Content-range: partial reviewal of the resource.
      • Content-type
      • Expires
      • Last Modified
